## Round 2 — CURSOR AGENT (DATA + LUOGO es. CASA o ACADEMY):

Practical recommendation
Start in the IDE editor for the skeleton
Create or open the project folder (File → New Project / Open Folder), initialize Git, pick your stack (create-next-app, cargo new, etc.), verify the terminal and toolchain. That stays the fastest way to guarantee a known-good baseline: dependencies install, scripts run, you can debug and click through errors.

Use Cursor 3–style workflows (Agents Window, parallel agents, worktrees) once the repo exists when you want:

Large bootstrap or migration bursts (lots of files, repeated tries).
Parallel attempts or isolated branches (changelog mentions /worktree and /best-of-n in the editor).
Work that fits agents running in parallel across environments more than typing in one file.
Prefer staying mostly in the editor when day one is about feel of the stack, debugging, extensions, tight edit–save–breakpoint loops, or anything where you want immediate control over every change.

SUMMARY

Lay down the project in the IDE, then branch into Agents Window / agent-heavy flows when the payoff (scale, parallelism, exploratory branches) outweighs wanting a minimal, hands-on workspace. Cursor’s changelog explicitly says you can switch back to the IDE anytime or use both.
