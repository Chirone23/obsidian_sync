## Quando applicarla
Ogni volta che l'utente descrive una funzionalità, un progetto o un problema tecnico e vuole sapere quali tool/librerie/framework usare.

## Procedura
1. Identifica il tipo di problema (frontend, backend, automazione, AI, mobile, database, ecc.)
2. Proponi 2-3 opzioni per ogni layer dello stack necessario
3. Per ciascuna opzione fornisci:
   - Nome e link alla documentazione ufficiale
   - Caso d'uso ideale
   - Pro e contro
   - Livello di complessità (Basso / Medio / Alto)
4. Indica la combinazione consigliata con motivazione
5. Se esiste un tutorial o esempio pratico rilevante, linkalo

## Output atteso
- Tabella opzioni per layer (Tool | Caso d'uso | Pro | Contro | Complessità)
- Raccomandazione finale con motivazione
- Link a documentazione e risorse pratiche

## Note
- Contesto di default: Flutter/React Native (mobile), Supabase/Firebase (backend), n8n (automazione)
- Preferisci stack open source o con piano gratuito generoso
- Segnala se una tecnologia è in rapida evoluzione o a rischio deprecazione
